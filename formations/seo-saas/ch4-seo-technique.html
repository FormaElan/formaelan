Chapitre 4

SEO technique pour SaaS

⏱ 45 min · 3 leçons
Le SEO technique d'un SaaS a ses propres pièges : des milliers de pages /app générées dynamiquement, des routes authentifiées que Google ne peut pas crawler, des Core Web Vitals sacrifiés pour des features produit. Ce chapitre couvre les points techniques qui font une vraie différence.
Figure 4.1 — Dashboard Core Web Vitals : LCP (vitesse chargement), INP (réactivité), CLS (stabilité visuelle). Objectif : les 3 dans le vert.

4.1 — Le piège des pages /app

La majorité des SaaS génèrent des milliers de pages derrière /app, /dashboard, /user, etc. Ces pages ne doivent jamais être indexées — mais elles consomment du crawl budget si elles ne sont pas bloquées correctement.

  • Robots.txt : Disallow: /app/ bloque le crawl mais pas l'indexation si des liens externes pointent dessus.
  • Meta noindex : plus sûr pour les pages authentifiées. Combine avec robots.txt pour maximiser la protection.
  • Canonical : pour les pages dynamiques avec paramètres (?sort=, ?filter=), utilise le canonical pour éviter le duplicate content.

4.2 — Structured Data pour les SaaS

  • SoftwareApplication : schema.org pour ton produit — permet d'apparaître dans les rich results avec note, prix, OS compatibles.
  • FAQ schema : sur tes pages de vente et landing pages SEO pour occuper plus de place dans les SERP.
  • HowTo schema : pour tes tutoriels et guides techniques — booste le CTR sur les requêtes TOFU.
  • Article schema : sur tous tes articles de blog avec datePublished et dateModified — indique la fraîcheur à Google.
Points clés — Chapitre 4
  • Bloque /app et toutes les pages authentifiées avec robots.txt + noindex
  • Cible LCP < 2.5s, INP < 200ms, CLS < 0.1 — mesure dans Search Console
  • Implémente SoftwareApplication schema sur ta homepage produit
  • Les images non compressées sont souvent la cause #1 du mauvais LCP
